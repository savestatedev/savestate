1M Tokens Isn't Enough: Why Giant Context Windows Are Still Losing Your Agent's Memory

NVIDIA, Anthropic, and Google all ship 1M token context windows. Research shows reliable recall drops below 50% past 256K. The amnesia problem isn't solved. It's just bigger.

NVIDIA just shipped Nemotron 3 with a 1 million token context window. Claude has it. Gemini has it. The pitch is simple: just load everything into context and your agent will never forget.

Except it doesn't work that way.

The Amnesia Problem at Scale

A new paper on arXiv (AgentRM) formally identifies what practitioners have been noticing for months: long-running agent sessions suffer from context degradation. As the window fills, the model's ability to recall earlier information decays. The researchers call it "amnesia," and it's not a bug in any single model. It's a fundamental limitation of attention mechanisms at scale.

Current approaches handle this poorly. Most frameworks either truncate history (losing context you might need) or crash when they exceed limits. Neither option is acceptable for production agents.

1M Tokens Doesn't Mean 1M Usable Tokens

Here's the uncomfortable truth: benchmarks of GPT-5.4 and Gemini 3.1 Pro show both models struggle past 256K tokens. GPT-5.4 loses 54% of its retrieval accuracy when scaling from 256K to 1M. Your agent can technically fit everything in the window, but it can't reliably find it when needed.

Think about what this means. You load a million tokens of context, confident your agent has access to everything. It hallucinates anyway because the relevant information from 800K tokens ago might as well not exist.

The context window is a bucket, not a brain. Bigger buckets still leak.

MCP Is Making It Worse

A post from Apideck this week highlights another pressure: Model Context Protocol integrations are eating your working memory. MCP tool definitions can burn 55,000+ tokens before your agent processes a single user message. Load everything upfront for maximum capability and you sacrifice reasoning capacity. Limit your integrations to preserve working memory and your agent loses access to tools it needs.

There's no free lunch. Context is a zero-sum resource, and modern agent architectures are demanding more of it than ever.

Context Engineering: The New Required Skill

The AI Agents Stack (2026 Edition) makes a bold claim: prompt engineering is dead. The new discipline is context engineering.

This means architecting exactly what information your agent sees on every call. What gets stuffed in-context? What gets retrieved on demand? What gets summarized? What gets dropped entirely?

These decisions used to be optional optimizations. Now they're load-bearing architectural choices. Get them wrong and your agent forgets critical information. Get them right and you're still fighting against fundamental model limitations.

Five Ways Memory Compression Fails

Indium Tech published a breakdown of how agent memory compression fails in practice:

  1. Early-but-relevant information gets pruned
  2. Costs grow non-linearly with context length
  3. Summarization loses critical details
  4. Retrieval systems return wrong context
  5. Failures happen silently with no error signal

That last one is the killer. Your agent doesn't tell you it forgot something. It just confidently proceeds with incomplete information.

The Case for External Memory

None of this means large context windows are useless. They're a genuine improvement. But they're not a replacement for proper memory architecture.
